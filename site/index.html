LinkPilot

把每个链接送到正确的浏览器、Profile 和 Workspace。它不是浏览器，而是站在 macOS、应用和浏览器之间的轻量路由层。

下载最新版本 查看 GitHub
macOS 桌面应用 后台 daemon lpt CLI 中英繁日界面

Why it exists

多浏览器工作流的麻烦，不该靠手动复制链接解决。

开发者、顾问和安全敏感用户经常同时使用 Chrome Work Profile、个人浏览器、 Arc、Safari、Firefox 和多个工作空间。LinkPilot 把“这个链接该去哪”变成可解释、 可测试、可维护的本地规则。

01

外部应用打开错浏览器

Slack、Lark、Raycast、Terminal 或 IDE 发出的链接，经常落到错误身份环境。

02

OAuth 回调混乱

登录流程被送进个人 Profile，或者公司账号回调进了测试浏览器。

03

浏览器 Profile 隔离难维护

工作、个人、客户账号混在一起，Cookie 边界靠记忆维持。

04

缺少可解释的决策记录

链接为什么去了那里，需要从规则和来源上下文里直接看见。

Routing engine

路由不是猜测，而是一条条可测试的规则。

LinkPilot 可以按 host、path、来源 App、来源浏览器和来源 Profile 判断目标。 没命中的链接走默认目标，需要人工选择时交给 Ask Picker。

What ships today

桌面、后台服务和命令行是同一个路由系统。

当前版本聚焦 macOS：桌面应用负责配置和可视化，daemon 负责后台路由， `lpt` CLI 负责脚本化操作和诊断。

A

桌面配置台

Rules、Browsers、Settings、Inspector 和 Test URL 页面覆盖日常配置、浏览器管理、 路由 dry-run 和实时决策检查。

B

后台 daemon

LinkPilot.app 关闭主窗口后，路由仍由后台服务继续处理。CLI 和 GUI 通过本机 Unix socket 与同一个服务通信。

C

`lpt` 命令行

在终端里打开 URL、管理规则、导入导出配置、安装 daemon、检查默认浏览器状态， 适合自动化和快速诊断。

Ask Picker

不确定时，让选择也足够快。

Ask 规则会打开 LinkPilot 的浏览器选择窗口。多 Profile 浏览器可以用 Halo 选择环快速瞄准 Profile，键盘位置 1 到 9 也能映射到常用目标。

Trust boundary

本地工具应该把边界讲清楚。

LinkPilot 的目标不是替代浏览器，也不是读取网页内容。它只关心链接、来源上下文、 用户配置和最终打开目标。

不做通用浏览器

LinkPilot 不渲染网页，只接收 URL 并交给真实浏览器或 Profile。

配置留在本机

规则和设置写入本地配置文件，daemon 通过本机 IPC 提供服务。

决策可解释

Inspector 和 Test URL 用真实路由引擎展示命中的规则与目标。

更新需校验

自动下载的 DMG 会按 release 中的 checksums.txt 校验 SHA-256。

Install

先用 DMG 或 CLI tarball 安装。

当前发布物未签名未公证，macOS 首次启动可能需要移除 quarantine 标记。 Homebrew 渠道还未作为正式安装路径发布。
