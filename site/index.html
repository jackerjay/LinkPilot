Why LinkPilot

多浏览器工作流缺的，是一层可维护的调度。

复制链接、切浏览器、找 Profile 是重复劳动；更糟的是 OAuth 回跳、客户账号和公司 Cookie 边界会被错误的打开方式打乱。LinkPilot 把这些决策收进一套本地规则。

👩‍💻

开发者

GitHub、Figma、Notion 进工作 Profile；个人站点留在日常浏览器；localhost 永远落在开发环境上下文。

Halo Picker

拿不准的链接？
按住 ⌥ Option，唤出 Halo 环。

Halo 是 LinkPilot 的径向选择环：链接打开瞬间按住 Option， 所有浏览器 Profile 围成一圈，移动指针或敲数字键直达目标。

  • 数字键 1–9 直达任意 Profile，肌肉记忆比鼠标更快。
  • 三种风格 —— Frosted 毛玻璃、Bezel 旋钮、Crown 鎏金，右侧可切换试戴。
  • 扇区支持直接拖拽重排，新 Profile 从保留的 + 扇区加入。

悬停扇区，或直接按 1–6 试试

lpt CLI

键盘党？
全部能力都在终端里。

lpt 覆盖 GUI 的全部配置面：规则、Workspace、浏览器、设置、默认目标。 写操作直达本地配置文件，运行中的 daemon 自动感知 —— 不需要重启任何东西。

  • lpt open 走完整路由引擎，和点击链接行为一致。
  • lpt doctor 一条命令体检 daemon、默认浏览器与配置。
  • 脚本友好：所有子命令支持 JSON 输出。

Install

30 秒装好，让链接自己认路。

推荐 Homebrew：cask 会自动放入 /Applications 并处理 quarantine 标记。 不想加 tap 也可以手动下载 DMG。 

$ brew tap jackerjay/linkpilot
$ brew install --cask jackerjay/linkpilot/linkpilot

# 构建未签名，cask 的 postflight 会自动移除 quarantine 标记；
# Homebrew 首次会提示信任该 tap，确认一次即可。

装完后在「系统设置 → 默认网页浏览器」里选择 LinkPilot，路由就生效了。

# Apple Silicon → aarch64；Intel → x86_64
$ ARCH=$(uname -m | sed "s/arm64/aarch64/")
$ curl -L "https://github.com/jackerjay/LinkPilot/releases/latest/download/LinkPilot_<version>_${ARCH}.dmg" -o LinkPilot.dmg
$ hdiutil attach LinkPilot.dmg
$ cp -R "/Volumes/LinkPilot/LinkPilot.app" /Applications/
$ hdiutil detach "/Volumes/LinkPilot"
$ xattr -dr com.apple.quarantine /Applications/LinkPilot.app
$ open /Applications/LinkPilot.app

发布物未签名未公证，下载后可对照 Release 页的 checksums.txt 校验 SHA-256。

$ brew install jackerjay/linkpilot/linkpilot-cli

# 安装 lpt + linkpilot-daemon 到 PATH，适合无 GUI 的纯键盘工作流。

CLI 与 GUI 共用同一份本地配置，可以随时混用。
