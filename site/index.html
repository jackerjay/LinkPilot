Why it exists

多浏览器工作流真正缺的是一层可维护的调度。

复制链接、切浏览器、找 Profile 是重复劳动；更糟的是 OAuth、客户账号和公司 Cookie 边界会被错误打开方式打乱。LinkPilot 把这些决策收进一套本地规则。

01

开发者

GitHub、Figma、Notion 进工作 Profile；个人站点留在日常浏览器；localhost 保持开发环境上下文。

02

多客户协作

按域名、来源应用和工作空间把链接送到对应客户身份，减少账号串台和误登录。

03

安全敏感用户

不读网页正文，不做远程遥测；每次路由为什么发生，都能在 Inspector 里回看。

Routing engine

规则覆盖 GUI、daemon 和 CLI，不是只做一个默认浏览器代理。

当前版本已经支持按 host、path、来源应用、来源浏览器和来源 Profile 路由 URL； 桌面界面、命令行和后台服务使用同一套配置与路由引擎。

URL matcher 按 host 与 path 匹配 GitHub、OAuth、Figma、Notion、localhost 等场景。
Source context 把 Slack、Lark、Terminal、IDE 或来源浏览器上下文纳入规则。
Browser inventory 识别 Chrome 系、Arc、Safari、Firefox 和自定义浏览器。
Local config 配置写入本地文件，daemon 通过 fsnotify 自动感知变化。
Example rule
when host = github.com
and from app = Slack
open Chrome / Work Profile
otherwise ask with Picker

Ask Picker

不确定的链接，也要能快速决定。

Trust boundary

LinkPilot 的边界是本地、可解释、少权限。

它不渲染网页，不读取网页正文，不把规则决策上传到远程服务。你配置的规则、默认目标、 浏览器清单和路由历史都围绕本机工作流设计。

不是浏览器

LinkPilot 只接收 URL 并交给真实浏览器。

配置在本机

规则写入本地配置文件，daemon 通过本机 IPC 服务 GUI 和 CLI。

结果可解释

Test URL 和 Inspector 展示真实路由决策。

更新需校验

下载的 DMG 会按 release checksums.txt 校验 SHA-256。

Install

当前推荐从 GitHub Release 安装。

目前发布物未签名未公证，macOS 首次启动可能需要移除 quarantine 标记。 Homebrew 渠道尚未作为正式安装路径发布。
