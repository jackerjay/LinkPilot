# LinkPilot v0.2.0 — Headless daemon + TypeScript config

This is the body for the v0.2.0 GitHub Release. `release.yml`'s
`generate_release_notes: true` will prepend the auto-generated commit
log; everything below is the human-curated summary.

## Highlights

- **A. Daemon split from GUI.** `linkpilot-daemon` runs as a macOS
  LaunchAgent and survives every GUI close — the CLI works fully
  without ever opening the app window.
- **B. TypeScript config DSL.** Write rules as code in
  `linkpilot.config.ts`, compile with `lpt config compile`. Published
  to npm as [`@linkpilot/config`][npm].
- **C. New CLI surface.** `lpt history`, `lpt daemon {start, stop,
  restart, status, install, uninstall, logs}`, `lpt config compile`.
- **D. Install via Homebrew.** `brew install --cask jackerjay/linkpilot/linkpilot`
  (GUI + CLI) or `brew install jackerjay/linkpilot/linkpilot-cli`
  (CLI only).
- **E. Menubar-only mode.** Raycast/Alfred style: no Dock icon, no
  Cmd+Tab presence, no focus-stealing. Tray icon + main window are
  the only UI surfaces.

## Breaking

- **Binary renamed `lp` → `lpt`** to avoid PATH collision with macOS's
  built-in `/usr/bin/lp` (CUPS). v0.1 users replace any `lp` in
  scripts / aliases / shell history with `lpt`. The shell symlink the
  GUI installs (Settings → "Install CLI on PATH") now points at
  `~/.local/bin/lpt`.
- **IPC protocol bumped 1 → 2.** Adds `RouteHistory` verb. v0.1
  clients ↔ v0.2 daemon: client gets a structured "unknown-verb"
  error + upgrade hint, doesn't silently break. v0.2 clients ↔ v0.1
  daemon: same hint surfaces on the CLI side via transport-close
  detection.
- **GUI is now an "agent" app** (LSUIElement=true). It will not appear
  in the Dock. Quit via the tray menu instead of Cmd+Q.
